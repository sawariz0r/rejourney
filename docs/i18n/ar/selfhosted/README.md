# الاستضافة الذاتية Rejourney

هذا الدليل مخصص لـ **أي شخص** الذي يقوم بتشغيل Rejourney على الخادم الخاص به (عادةً VPS واحد أو جهاز مخصص) باستخدام مكدس **Docker Compose** الرسمي. لا تحتاج إلى الوصول إلى البنية التحتية الداخلية لـ Rejourney أو Kubernetes.

بعد الإعداد تحصل على:

- **لوحة القيادة على شبكة الإنترنت** في نطاقك (HTTPS عبر Let’s Encrypt)
- **API** على مجال فرعي (للوحة المعلومات والهاتف المحمول SDK)
- **استيعاب (تحميل) التتابع** على نطاق فرعي آخر (تمر عمليات تحميل الجلسة عبر الخادم الخاص بك، وليس مباشرة من الهواتف إلى وحدة تخزين الكائنات)
- **PostgreSQL** و **Redis** وإما **المدمج في MinIO** أو **تخزين S3-compatible الخاص بك**
- الخلفية **العمال** التي تعالج الجلسات والاحتفاظ والتنبيهات (نفس الأدوار كما في النشر السحابي لـ Rejourney)

تفترض جميع الأوامر أدناه أنك في **جذر المستودع** بعد الاستنساخ (المجلد الذي يحتوي على `docker-compose.selfhosted.yml`).

## ما تحتاجه مسبقا

### الخادم

- **نظام التشغيل:** Ubuntu 22.04+، Debian 12+، أو Linux آخر يقوم بتشغيل Docker بشكل جيد
- **Docker:** 24 ​​أو أحدث، مع **البرنامج المساعد Docker Compose** (يجب أن يعمل `docker compose version`)
- **الموارد (مستحسن):** 4 vCPU، ذاكرة الوصول العشوائي (RAM) سعة 8 جيجابايت، قرص سعة 40 جيجابايت (المزيد إذا احتفظت بالعديد من التسجيلات)
- منفذا **شبكة:** **80** و **443** مفتوحان على الإنترنت (مطلوبان لتحدي Let’s Encrypt HTTP وHTTPS)

### المجال وDNS

أنت بحاجة إلى **مجال أساسي واحد** الذي تتحكم فيه (على سبيل المثال `example.com`). قبل تشغيل برنامج التثبيت، قم بإنشاء سجلات DNS **أ** (أو **AAAA**) تشير إلى **الجميع** لأسماء المضيفين هذه على عنوان IP العام لخادمك:

| اسم المضيف | الغرض |
|----------|---------|
| `example.com` | لوحة القيادة |
| `www.example.com` | يعيد التوجيه إلى لوحة القيادة |
| `api.example.com` | API (و WebSocket عند استخدامها) |
| `ingest.example.com` | ترحيل التحميل (يستخدم SDK هذا تلقائيًا بمجرد تكوين API) |

استبدل `example.com` بنطاقك الحقيقي. يمكن أن يستغرق الانتشار بضع دقائق إلى ساعات؛ لن يتم إصدار شهادات TLS حتى يتم حل DNS بشكل صحيح.

### Let’s Encrypt

سيُطلب منك **عنوان البريد الإلكتروني** أثناء التثبيت. يتم استخدامه لإشعارات انتهاء صلاحية الشهادة من Let’s Encrypt.

### الأدوات الموجودة على جهازك

- `git` لاستنساخ المستودع
- `openssl` (يستخدمه البرنامج النصي للتثبيت لإنشاء الأسرار)
- الصدفة (bash جيدة)

## التثبيت لأول مرة

### 1. استنساخ المستودع

```bash
git clone https://github.com/rejourneyco/rejourney.git
cd rejourney
```

ابق على الفرع الافتراضي (أو علامة الإصدار إذا كان المشروع يوثق واحدًا للاستضافة الذاتية).

### 2. قم بتشغيل برنامج التثبيت

```bash
./scripts/selfhosted/deploy.sh install
```

البرنامج النصي سوف:

1. اطلب **المجال الأساسي** (على سبيل المثال، `example.com` - وليس `https://`، لا يوجد مسار).
2. اطلب **البريد الإلكتروني Let’s Encrypt** الخاص بك.
3. اطلب **تخزين**: تخزين **MinIO** المدمج (مستحسن) أو تخزين **خارجي S3-compatible** (سوف تقوم بإدخال نقطة النهاية والدلو والمنطقة والمفاتيح).
4. قم بإنشاء **`.env.selfhosted`** في جذر الريبو باستخدام كلمات المرور والأسرار التي تم إنشاؤها. يتم تطبيق **تقييد الأذونات** (`chmod 600`).
5. قام **يحذب** بنشر صور الحاويات (API، الويب، العمال، قواعد البيانات، Traefik، وما إلى ذلك).
6. **يبني** صورة **التمهيد / الهجرة** **من استنساخك** (تحتوي على البرامج النصية لإعداد قاعدة البيانات؛ ولم يتم تنزيلها من سجل الحاوية).
7. ابدأ تشغيل قواعد البيانات، Redis، وTraefik، و(إذا تم اختياره) MinIO.
8. التحقق من صحة اتصال قاعدة البيانات باستخدام `DATABASE_URL` الذي تم تكوينه قبل تشغيل bootstrap.
9. قم بتشغيل حاوية **bootstrap** لمرة واحدة: مخطط قاعدة البيانات، والبذور الاختيارية لأول مرة، وتكوين التخزين في قاعدة البيانات.
10. ابدأ تشغيل API، وقم بتحميل المرحل ولوحة المعلومات والعاملين.

يمكن أن يستغرق التثبيت الأول عدة دقائق (عمليات سحب الصور والتمهيد).

### 3. حماية `.env.selfhosted`

يحتوي هذا الملف على **كل الأسرار** للنشر (قاعدة البيانات، Redis، JWT، تشفير التخزين، بيانات اعتماد MinIO إذا تم استخدامها، وما إلى ذلك). **قم بعمل نسخة احتياطية منه** إلى مكان آمن (مدير كلمات المرور، النسخ الاحتياطي المشفر). إذا فقدتها، فقد تفقد القدرة على فك تشفير بيانات الاعتماد المخزنة أو إعادة إنشاء نفس عملية النشر.

لا تلزمه بـ git (يجب تجاهله بواسطة `.gitignore`).

## بعد التثبيت

### عناوين URL

يقوم المثبت بطباعة عناوين URL. على العموم:

- **لوحة القيادة:** `https://<your-base-domain>`
- **API:** `https://api.<your-base-domain>`
- **استيعاب:** `https://ingest.<your-base-domain>`

يقوم `www.<your-base-domain>` بإعادة التوجيه إلى لوحة المعلومات.

### التحقق من المكدس

```bash
./scripts/selfhosted/deploy.sh status
```

يجب أن تشاهد الحاويات قيد التشغيل؛ يجب أن يصبح `api` و`ingest-upload` **صحيح** بعد وقت قصير.

### تسجيل الدخول الأول واختبار التسجيل

1. افتح لوحة التحكم في المتصفح.
2. إنشاء حساب ومشروع.
3. قم بتكوين Rejourney SDK لتطبيقك باستخدام **عنوان URL API** (راجع [تكوين SDK](#configuring-your-mobile-app) أدناه).
4. سجل جلسة قصيرة وتأكد من ظهورها في إعادة التشغيل.

إذا لم تظهر الجلسات مطلقًا في إعادة التشغيل، فراجع [استكشاف الأخطاء وإصلاحها](/docs/selfhosted/troubleshooting) (تحميل التتابع واستيعاب سجلات العامل).

## العمليات اليومية

كل هذه تعمل من جذر الريبو.

| العمل | الأمر |
|--------|---------|
| حالة الخدمة | `./scripts/selfhosted/deploy.sh status` |
| متابعة كافة السجلات | `./scripts/selfhosted/deploy.sh logs` |
| سجلات لخدمة واحدة | `./scripts/selfhosted/deploy.sh logs api` (استبدل `api` بـ `web`، `ingest-upload`، `ingest-worker`، وما إلى ذلك) |
| صور **يرقي** وأعد تشغيل التمهيد | `./scripts/selfhosted/deploy.sh update` |
| أوقف كل شيء عن حذف البيانات **بدون** | `./scripts/selfhosted/deploy.sh stop` |
| حاويات وأحجام **إعادة ضبط** (اتلافية) | `./scripts/selfhosted/deploy.sh reset` |

يسحب **`update`** صورًا أحدث (حيثما أمكن)، ويعيد بناء صورة التمهيد من النسخة الحالية، ويعيد تشغيل المكدس، ويقوم بتشغيل التمهيد مرة أخرى بحيث يظل مخطط قاعدة البيانات وإعدادات التخزين متوافقة مع `.env.selfhosted`. يقوم بمسح **لا** Postgres أو وحدات تخزين الكائنات.

قبل التمهيد، يقوم كل من `install` و`update` بالتحقق من صحة اتصال قاعدة البيانات باستخدام بيانات الاعتماد التي تم تكوينها. إذا لم تتطابق بيانات الاعتماد مع بيانات Postgres المستمرة، فسيتوقف النشر مبكرًا باستخدام إرشادات الاسترداد بدلاً من الفشل لاحقًا في التمهيد.

**`stop`** يوقف الحاويات فقط؛ تظل Docker **مجلدات** (بيانات Postgres، بيانات MinIO، وما إلى ذلك) حتى تقوم بإزالتها بشكل صريح.

يقوم **`reset`** بإزالة الحاويات ذاتية الاستضافة ووحدات تخزين Docker (`pgdata`، `redisdata`، `miniodata`، `traefik-certs`) بعد مطالبة التأكيد. كما أنه يمزق حاويات ملف التعريف MinIO حتى في حالة فقدان `.env.selfhosted`، لذا فإن بيانات MinIO القديمة لا تمنع التثبيت التالي. استخدم هذا فقط عندما تريد تثبيتًا جديدًا تمامًا.

## التخزين: MinIO مقابل S3 الخارجي

### المدمج في MinIO (افتراضي)

- الأسهل بالنسبة لخادم واحد: يتم تشغيل تخزين الكائنات على **داخل Docker** ولا يتم عرضه على الإنترنت العام افتراضيًا.
- تتم كتابة بايتات الجلسة بواسطة خدمة **استيعاب وتحميل**؛ لا تحتاج الأجهزة إلى الوصول إلى MinIO مباشرة.
- تتم معالجة إنشاء الجرافة أثناء التثبيت.

### وحدة تخزين خارجية S3-compatible

استخدم AWS S3 أو Cloudflare R2 أو Hetzner Object Storage أو Wasabi أو أي S3-compatible API. أثناء التثبيت، يمكنك توفير عنوان URL لنقطة النهاية والحاوية والمنطقة ومفاتيح الوصول.

أمثلة على أنماط عنوان URL لنقطة النهاية (مستندات موفر الخدمة لديك موثوقة):

- AWS: `https://s3.<region>.amazonaws.com`
- Cloudflare R2: `https://<account-id>.r2.cloudflarestorage.com`
- هيتزنر: `https://<location>.your-objectstorage.com`

إذا قمت بإضافة **عنوان URL عام منفصل** للتنزيلات، فقم بتعيين `S3_PUBLIC_ENDPOINT` في `.env.selfhosted` وقم بتشغيل `./scripts/selfhosted/deploy.sh update`.

## التكوين المهم (`.env.selfhosted`)

يقوم المثبت بإنشاء هذا الملف. تشمل المتغيرات النموذجية ما يلي:

- **النطاقات وعناوين URL العامة:** `BASE_DOMAIN`، `DASHBOARD_DOMAIN`، `API_DOMAIN`، `INGEST_DOMAIN`، `PUBLIC_*_URL`
- **قاعدة البيانات:** `DATABASE_URL` (يشير إلى خدمة `postgres` داخل Compose)
- **Redis:** `REDIS_URL`
- **تخزين:** `STORAGE_BACKEND`، `S3_*`، واختياريًا `MINIO_*`
- **حماية:** `JWT_SECRET`، `JWT_SIGNING_KEY`، `INGEST_HMAC_SECRET`، `STORAGE_ENCRYPTION_KEY`

عمليات التكامل الاختيارية (اتركها فارغة إذا لم يتم استخدامها): Stripe، SMTP، GitHub OAuth، إلخ.

**تغيير مساحة التخزين أو القيم المتعلقة بالمجال:** قم بتحرير `.env.selfhosted`، ثم قم بتشغيل:

```bash
./scripts/selfhosted/deploy.sh update
```

## كيفية عمل إعداد قاعدة البيانات (التمهيد الأول مقابل التحديثات اللاحقة)

عادةً ما تحتاج **لا** إلى تشغيل SQL يدويًا. تتعامل معها الحاوية **bootstrap**.

- **قاعدة بيانات فارغة جديدة تمامًا:** يطبق المكدس المخطط الحالي من التعليمات البرمجية، ثم يسجل إصدارات الترحيل التي تم استيفاءها بالفعل، لذا فإن التحديثات المستقبلية تطبق فقط عمليات الترحيل **جديد**.
- **قاعدة البيانات الموجودة (تمت تهيئتها بالفعل):** يتم تطبيق عمليات الترحيل **قيد الانتظار** فقط. لا تتم إعادة بناء بياناتك من الصفر على كل `update`.
- إذا كانت قاعدة البيانات **لديها بالفعل الجداول** ولكن جدول محفوظات الترحيل هو **مفقودة أو فارغة** (على سبيل المثال استعادة جزئية)، فقم بتمهيد **يتوقف مع وجود خطأ** لتجنب التلف العرضي. تم توثيق خيارات الاسترداد المتقدمة في [استكشاف الأخطاء وإصلاحها](/docs/selfhosted/troubleshooting).

## خوادم Apple Silicon وARM

على أجهزة **أرم64** (العديد من أجهزة Mac، وبعض المثيلات السحابية)، يقوم البرنامج النصي للنشر بتعيين `DOCKER_DEFAULT_PLATFORM=linux/amd64` لسحب الصور عندما لا تقوم بتعيينها بنفسك، لذلك لا تزال الصور المعدة مسبقًا التي تنشر `amd64` فقط قيد التشغيل. إذا كنت بحاجة إلى سلوك مختلف، فقم بتعيين `DOCKER_DEFAULT_PLATFORM` في بيئتك قبل تشغيل البرنامج النصي.

صورة **bootstrap** تكون دائمًا **بنيت على جهازك** من المستودع المستنسخ، لذلك فهي دائمًا تطابق عملية الدفع الخاصة بك.

## ما يتم تشغيله في Docker (نظرة عامة)

- شهادات **Traefik:** HTTPS والتوجيه إلى لوحة المعلومات، API، واستيعاب أسماء المضيفين.
- **Postgres / Redis:** بيانات التطبيق وقوائم الانتظار.
- **MinIO:** تخزين كائن داخلي اختياري.
- **API:** الرئيسي HTTP API.
- **الاستيعاب والتحميل:** خدمة مخصصة لتحميل حركة ترحيل البيانات.
- **الويب:** واجهة المستخدم الثابتة للوحة القيادة.
- **العمال:** معالجة قوائم انتظار الاستقبال، وإعادة تشغيل العناصر، ودورة حياة الجلسة، وعمل نمط الاحتفاظ المجدول، والتنبيهات.

يوجد عامل دفعة فواتير منفصل **لا** في هذه المكدس؛ يتم تشغيل تكامل الفوترة بواسطة Stripe وAPI عند تكوين المفاتيح.

## تكوين تطبيق الهاتف المحمول الخاص بك

قم بتوجيه SDK إلى مضيف **لك** API (يجب أن يتطابق مع `API_DOMAIN` / `PUBLIC_API_URL`).

### مثال React Native

```ts
import { initRejourney, startRejourney } from '@rejourneyco/react-native';

initRejourney('pk_live_your_public_key', {
  apiUrl: 'https://api.example.com',
});

startRejourney();
```

استخدم عنوان URL الحقيقي لـ API. يتم اشتقاق عناوين URL للتحميل لـ `ingest.<your-domain>` تلقائيًا عندما يتم تكوين الخادم بشكل صحيح.

## النسخ الاحتياطية

على الأقل، قم بعمل نسخة احتياطية من **PostgreSQL** و **`.env.selfhosted`** و(إذا كنت تستخدم MinIO) **بيانات تخزين الكائن** المضمن.

```bash
./scripts/selfhosted/backup.sh
./scripts/selfhosted/backup.sh --full
```

التفاصيل: [النسخ الاحتياطي والاسترداد](/docs/selfhosted/backup-recovery).
